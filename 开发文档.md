# 合同模板管理系统 - 产品需求文档 (PRD)

> **文档版本**: v1.0
> **创建日期**: 2026-06-09
> **技术栈**: React + TypeScript / Python3 + FastAPI

---

## 一、项目背景与目标

### 1.1 项目背景

企业在日常运营中需要处理大量合同，合同模板管理混乱、版本追踪困难、审批流程不透明是普遍痛点。本系统旨在解决合同模板的全生命周期管理问题。

### 1.2 核心目标

- 实现合同模板的标准化管理
- 支持模板版本控制与历史追溯
- 提供灵活的变量填充机制
- 实现合同审批流程自动化

---

## 二、用户痛点分析

| 痛点编号 | 痛点描述 | 影响程度 | 解决方案方向 |
|---------|---------|---------|-------------|
| P1 | 模板散落在各处，查找困难 | 高 | 统一模板库 + 分类标签 |
| P2 | 模板版本混乱，不知道哪个是最新的 | 高 | 版本控制 + 主版本标记 |
| P3 | 合同内容重复填写，效率低 | 高 | 变量模板 + 自动填充 |
| P4 | 审批流程不透明，进度难追踪 | 中 | 审批工作流 + 状态看板 |
| P5 | 合同生成后格式错乱 | 中 | 模板预览 + 格式校验 |
| P6 | 缺乏合同归档和检索能力 | 中 | 合同档案库 + 全文搜索 |

## 三、功能需求拆解

### 3.1 模板管理模块

#### 功能清单

| 功能点 | 优先级 | 描述 |
|-------|-------|------|
| 模板创建 | P0 | 支持上传 Word/PDF 模板，在线编辑 |
| 模板分类 | P0 | 多级分类树，支持自定义标签 |
| 模板搜索 | P0 | 关键词搜索 + 分类筛选 |
| 模板预览 | P0 | 在线预览模板内容 |
| 版本管理 | P1 | 保存历史版本，支持回滚 |
| 主版本标记 | P1 | 标记当前生效版本 |

#### 数据结构设计

```python
# Template 模板表
class Template:
    id: str                    # UUID
    name: str                  # 模板名称
    category_id: str           # 分类ID
    tags: list[str]            # 标签列表
    content: str               # 模板内容（存储路径）
    variables: list[Variable]  # 变量定义
    current_version: str       # 当前版本号
    status: str                # draft/active/archived
    created_at: datetime
    updated_at: datetime

# TemplateVersion 版本表
class TemplateVersion:
    id: str                    # UUID
    template_id: str           # 关联模板ID
    version_number: str        # 版本号 (如 v1.0, v1.1)
    content: str               # 版本内容路径
    change_log: str            # 变更说明
    is_master: bool            # 是否为主版本
    created_by: str            # 创建人
    created_at: datetime
```

```

循环变量: {{#列表}}...{{/列表}}

```

### 4.5 核心API设计

#### 模板管理 API

```yaml
# 模板列表
GET /api/v1/templates
  Query: page, page_size, category_id, keyword, status
  Response: { items: Template[], total: int, page: int }

# 创建模板
POST /api/v1/templates
  Body: { name, category_id, tags[], file }
  Response: Template

# 获取模板详情
GET /api/v1/templates/{template_id}
  Response: Template

# 更新模板
PUT /api/v1/templates/{template_id}
  Body: { name, category_id, tags[] }
  Response: Template

# 删除模板
DELETE /api/v1/templates/{template_id}
  Response: { success: bool }

# 获取模板版本列表
GET /api/v1/templates/{template_id}/versions
  Response: TemplateVersion[]

# 设置主版本
PUT /api/v1/templates/{template_id}/versions/{version_id}/set-master
  Response: TemplateVersion
```

#### 合同生成 API

```yaml
# 解析模板变量
GET /api/v1/templates/{template_id}/variables
  Response: Variable[]

# 预览合同
POST /api/v1/contracts/preview
  Body: { template_id, variables: { key: value } }
  Response: { preview_url: string }

# 生成合同
POST /api/v1/contracts
  Body: { template_id, variables: { key: value }, title }
  Response: Contract

# 导出合同
GET /api/v1/contracts/{contract_id}/export
  Query: format (word/pdf)
  Response: File

# 批量生成
POST /api/v1/contracts/batch
  Body: { template_id, excel_file }
  Response: { task_id: string }

# 查询批量任务状态
GET /api/v1/contracts/batch/{task_id}
  Response: { status, progress, result_url }
```

#### 审批流程 API

```yaml
# 发起审批
POST /api/v1/approvals
  Body: { contract_id, approver_ids[] }
  Response: Approval

# 获取待审批列表
GET /api/v1/approvals/pending
  Response: Approval[]

# 审批操作
POST /api/v1/approvals/{approval_id}/action
  Body: { action: approve/reject/transfer, comment, transfer_to? }
  Response: Approval

# 获取审批记录
GET /api/v1/approvals/{approval_id}/records
  Response: ApprovalRecord[]
```

### 5.2 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) DEFAULT 'user',
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 分类表
CREATE TABLE categories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    parent_id UUID REFERENCES categories(id),
    sort_order INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 模板表
CREATE TABLE templates (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,
    category_id UUID REFERENCES categories(id),
    tags JSONB DEFAULT '[]',
    description TEXT,
    status VARCHAR(20) DEFAULT 'draft',
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 模板版本表
CREATE TABLE template_versions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    template_id UUID REFERENCES templates(id) ON DELETE CASCADE,
    version_number VARCHAR(20) NOT NULL,
    file_path VARCHAR(500) NOT NULL,
    variables JSONB DEFAULT '[]',
    is_master BOOLEAN DEFAULT false,
    change_log TEXT,
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(template_id, version_number)
);

-- 合同表
CREATE TABLE contracts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(200) NOT NULL,
    template_id UUID REFERENCES templates(id),
    template_version_id UUID REFERENCES template_versions(id),
    variables JSONB NOT NULL,
    file_path VARCHAR(500),
    file_path_pdf VARCHAR(500),
    status VARCHAR(20) DEFAULT 'draft',
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 审批表
CREATE TABLE approvals (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    contract_id UUID REFERENCES contracts(id),
    status VARCHAR(20) DEFAULT 'pending',
    current_step INT DEFAULT 1,
    total_steps INT NOT NULL,
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 审批记录表
CREATE TABLE approval_records (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    approval_id UUID REFERENCES approvals(id) ON DELETE CASCADE,
    step INT NOT NULL,
    operator_id UUID REFERENCES users(id),
    action VARCHAR(20) NOT NULL,
    comment TEXT,
    transfer_to UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_templates_category ON templates(category_id);
CREATE INDEX idx_templates_status ON templates(status);
CREATE INDEX idx_contracts_status ON contracts(status);
CREATE INDEX idx_contracts_created_by ON contracts(created_by);
CREATE INDEX idx_approvals_status ON approvals(status);
CREATE INDEX idx_template_versions_master ON template_versions(template_id, is_master);
```

### 9.1 变量解析正则表达式

```python
import re

# 简单变量
SIMPLE_VAR_PATTERN = r'\{\{(\w+)\}\}'

# 带默认值
DEFAULT_VAR_PATTERN = r'\{\{(\w+)\|([^}]+)\}\}'

# 带类型校验
TYPED_VAR_PATTERN = r'\{\{(\w+):(\w+)(?::([^}]+))?\}\}'

# 循环变量
LOOP_VAR_PATTERN = r'\{\{#(\w+)\}\}(.*?)\{\{/\1\}\}'

# 条件变量
CONDITION_VAR_PATTERN = r'\{\{\?(\w+)\}\}(.*?)\{\{/\1\}\}'
```

### 9.2 Docker Compose 配置示例

```yaml
version: '3.8'

services:
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql+asyncpg://user:pass@db:5432/contract
      - REDIS_URL=redis://redis:6379/0
      - MINIO_ENDPOINT=minio:9000
    depends_on:
      - db
      - redis
      - minio

  frontend:
    build: ./frontend
    ports:
      - "3000:80"
    depends_on:
      - backend

  db:
    image: postgres:15-alpine
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=pass
      - POSTGRES_DB=contract
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    environment:
      - MINIO_ROOT_USER=admin
      - MINIO_ROOT_PASSWORD=admin123
    volumes:
      - minio_data:/data
    ports:
      - "9000:9000"
      - "9001:9001"

volumes:
  postgres_data:
  redis_data:
  minio_data:
```
