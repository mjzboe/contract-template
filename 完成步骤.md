# 工程师项目题完成步骤

## 题目核心

为律所 IPO 签字页业务构建一个**最小可用全栈系统**，核心闭环：**创建项目 → 选择模板 → 变量去重合并 → 填写变量 → 生成签字页 → 下载**。7 天内完成，建议 14 小时以内。

---

## 第一步：理解业务（~1h）

1. **理解签字页业务**：IPO 项目中，同一批股东/董监高/律师信息在几十份签字页模板中重复出现，需一次填写、全局复用
2. **理解核心业务对象**：
   - **签字页项目**：一次 IPO 项目的签字页整理任务
   - **模板/变量**：DOCX 模板中用 `【变量名】` 标记的占位符（如 `【张三】`、`【】年【】月【】日`）
   - **生成结果**：变量填充后生成的签字页文件（PDF/DOCX）
3. **理解核心流程**：创建项目 → 选择模板 → 系统自动提取并去重合并变量 → 用户填写变量（页面表单或 Excel 导入）→ 异步批量生成 → 下载

---

## 第二步：技术选型与架构设计（~1h）

1. **确定技术栈**（已有开发文档建议）：
   - 前端：React + TypeScript + Ant Design
   - 后端：Python + FastAPI
   - 数据库：PostgreSQL（或 SQLite 简化）
   - 文件存储：MinIO（或本地文件系统简化）
   - 文档渲染：docxtpl（Word 模板渲染）
2. **设计数据模型**：项目表、模板表、变量表、生成记录表
3. **设计 API 接口**：项目 CRUD、模板管理、变量提取/去重、生成与下载
4. **做取舍决策**：哪些简化（如 PDF 预览可简化为下载后查看、角色分配可 mock、认证可简化）

---

## 第三步：搭建项目骨架（~1.5h）

1. 初始化后端 FastAPI 项目，配置数据库连接
2. 创建数据库表结构（Alembic 迁移或直接建表）
3. 初始化前端 React 项目（Vite + TypeScript + Ant Design）
4. 配置前后端联调（CORS、代理）
5. 准备 DOCX 模板样例文件（根据 PDF 中的样例1、样例2 创建）

---

## 第四步：实现后端核心功能（~4h）

1. **模板管理**：上传 DOCX 模板、存储、列表查询
2. **变量提取**：用正则从 DOCX 中提取 `【变量名】` 格式的变量
3. **变量去重合并**：多个模板的变量聚合，同名变量只保留一个定义，一次填写全局复用
4. **项目 CRUD**：创建签字页项目、关联模板、保存变量值
5. **文档生成**：用 docxtpl 将变量值填充到模板，生成 DOCX
6. **下载接口**：返回生成的文件流
7. **Excel 批量导入**（可选）：解析 Excel 填入变量

---

## 第五步：实现前端核心功能（~4h）

1. **项目列表页**：展示已创建的签字页项目
2. **创建项目页**：输入项目名称，选择模板（多选）
3. **变量填写页**：展示去重后的变量列表，动态生成表单输入框
4. **生成与下载页**：触发生成、查看生成结果、下载文件
5. **Excel 导入功能**（可选）：上传 Excel 批量填入变量

---

## 第六步：联调与测试（~1.5h）

1. 前后端接口联调，确保数据流闭环
2. 用样例模板跑通完整流程：创建项目 → 选模板 → 填变量 → 生成 → 下载
3. 修复关键 Bug
4. 编写 README（本地复现步骤）

## 关键提醒

- **核心评估点**：① 能跑通最小闭环 ② 能讲清取舍 ③ 能说清 AI 使用方式与边界
- **变量语法**：模板中使用 `【变量名】` 格式（方括号），不是开发文档中的 `{{变量名}}`，需要根据实际模板调整解析逻辑
- **变量去重是核心亮点**：多个模板中同名变量只填一次，全局复用——这是解决"重复劳动"痛点的关键
- **14 小时上限**，超时反而扣分，合理控制边界
